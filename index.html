SwiftUI Stack Views 图解

基于 Apple《Building layouts with stack views》：用 HStack、VStack、ZStack 拆解复杂界面，并通过 alignment、Spacer、Divider、overlay/background 做出可适配布局。

1. 三种 Stack 的基本方向

Stack 的本质是把子视图放进一个简单的布局规则里：水平、垂直、或深度叠放。默认情况下，Stack 会居中对齐子视图，并在子视图之间加入少量间距。

2. 先规划布局层级

文档里的关键思想是：不要把复杂 UI 当成一个大布局来写，而是拆成小块。这个资料卡片可以拆成一个底层图片、一个底部覆盖栏、覆盖栏内部再横向和纵向组合。

HStack：底部信息栏横向展开
VStack + Spacer：文字上下排列，Spacer 推开剩余空间

3. alignment 控制的是 Stack 内部

VStack(alignment: .leading) 不是把整个 VStack 放到父容器左侧，而是让 VStack 里面的子视图沿水平方向左对齐。

对应地，HStack 的 alignment 控制子视图的垂直对齐；ZStack 的 alignment 可以同时表达水平和垂直方向。

VStack(alignment: .leading) {
    Text("Rachael Chiseck")
    Text("Chief Executive Officer")
}

4. Spacer 沿主轴占用剩余空间

在 HStack 里，Spacer 沿水平方向扩张，把左右内容推开。在 VStack 里，它沿垂直方向扩张，把上下内容推开。

Left
Right
HStack {
    Text("Left")
    Spacer()
    Text("Right")
}

5. Divider 是线，不是弹性空间

Divider 会在 Stack 的子视图之间插入一条分隔线。它只需要足够空间来画线，不会像 Spacer 一样吞掉剩余空间。

A
B
HStack {
    Text("A")
    Divider()
    Text("B")
}

6. 优先做自适应布局，而不是写死位置

Apple 文档强调：尽量描述结构和层级，让视图自然适配不同设备。写死宽高和坐标短期看起来能对齐，但在小屏、大字体、横竖屏或不同平台上更容易坏。

写死尺寸：容易溢出
固定宽度 210，放进窄屏后超出边界
自适应结构：跟随容器
使用 Stack、padding、Spacer，让内容适配可用空间

7. ZStack 与 overlay/background 的取舍

如果有一个主视图决定最终大小，附加内容只是贴在它上面，用 overlay 或 background 更自然。如果多个叠放视图共同决定整体大小，用 ZStack 更合适。
